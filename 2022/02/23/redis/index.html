1.Redis 基础

1.1 Redis 的存储结构

  • 字符串类型（String）
  • 字典（Hash）
  • 列表（List）
  • 有序集合（Zset）

1.2 Redis的基本命令

  • 判断一个键是否存在

  • 删除一个键

1.3 字符串（String）

字符串是最基本的数据类型，它能存储任何形式字符串，包括二进制数据。一个字符串类型键存储数据的最大容量是512MB。

1.3.1 基本命令

  • 设置/获取键值

  • 递增数字

  • 向尾部追加值

  • 获取字符串的长度

1.4 散列类型（Hash）

散列类型适合存储对象：使用key构成键名，使用field表示对象的属性。但field只能是字符串类型

1.4.1 基本命令

  • 设置/获取键值
  • 判断一个字段是否存在
  • 字段不存在才赋值
  • 增加数字
  • 删除字段
  • 获取键的所有字段名

1.5 列表类型

列表内部是一个双向列表，所在获取首尾两端的记录效率很高。但想要获取中间部分的数据就很慢

1.5.1 基本命令

  • 向列表中添加元素
  • 弹出元素
  • 获取列表长度
  • 获取列表的某一片段(包括两端)
  • 删除列表中指定的值
  • 通过索引操作键值
  • 保留列表指定片段内容
  • 在列表指定位置附近插入

1.6 集合类型(Set)

集合类型的特点是没有顺序，且元素唯一。

1.6.1 基本命令

  • 向集合中添加/删除元素
  • 获取集合中的所有元素
  • 判断元素是否在集合中
  • 集合间的运算
  • 集合中元素的个数

1.7 有序集合（Zset）

1.7.1基本命令

  • 增加/删除元素
  • 获取元素的分数
  • 元素的数量
  • 查看指定区间内元素的个数
  • 查看元素的排名

2.1 Redis的事务

2.1.1 事务的使用

保证了一个事务的所有的命令要么都执行，要么都不执行；并且不会被其他命令插入。

如果有一个命令发生语法错误，如set写成sat,则所有的命令都不会执行；当发生运行错误时，如：用散列类型的命令操作集合类型，这时，redis在执行之前无法发现。于是，只有这一条命令不执行，其他正确的命令依然执行。

Redis中没有回滚功能，因此开发者必须自己恢复。

在一个事务中当所有命令都依次执行完之后，才能得到每个结果的返回值。目前。后续命令无法使用前一条命令的返回结果

2.1.2 WATCH命令的使用

watch 命令可以监视一个或多个键，一旦被修改，则之后的事务不会被执行（用来防止静态条件）

3.Redis持久化

Redis 有两种持久化的方式：RDB和AOF

3.1 RDB

RDB是通过快照snapshotting方式完成的。当出现以下情况时，Redis会自动将内存中的所有数据生成一份副本并存储在硬盘上。

3.1.1 触发快照条件

  • 用户自定义配置规则进行自动快照

    这个规则有两个参数：时间窗口M和改动的键的个数N

    可以同时存在多个条件，这些条件以“或”形式存在。

  • 用户主动执行save 或BGSAVE命令

    save命令：Redis同步地进行快照操作，在快照执行过程中会阻塞所有来自客户端地请求。这可能会导致客户端长时间不响应。

    BGSAVE命令：后台异步执行快照操作，此时仍能响应客户端地请求。

  • 执行FLUSHALL命令

    此时只要定义了自动快照条件，在清空数据库时，Redis就会执行一次快照操作。

  • 执行复制时

    当设置了主从模式时，Redis会在复制初始化时进行自动快照。

3.1.2 快照原理

  1. Redis fork 当前进程，生成一个子进程
  2. 父进程继续接受并处理来自客户端地命令，子进程开始将内存中地数据写入硬盘中地临时文件。
  3. 子进程写入完毕后，用临时文件替换旧的RDB文件（dump.rdb）。

3.2 AOF

AOF 会将每一条写命令追加到硬盘文件(appendonly.aof)中。

但随着执行命令地越来越多，AOF文件中可能有很多冗余的命令。因此，应该设置一定的条件来避免这种情况。

或者可以通过BGREWRITEAOF命令来手动执行AOF重写。

注意：虽然开启了AOF，但由于操作系统的缓存机制，数据并没有真正地写入硬盘，而是进入了系统地硬盘缓存。在默认情况下，系统没30s进行一次同步操作，便将缓存中地内容真正写入硬盘。

此时可以开启aapendfsync参数，默认1s进行一次同步操作

4.Redis的集群

4.1 Redis的复制

通过复制，可以实现当一台数据库中的数据更新之后，自动将更新的数据同步到其他数据库上。

在复制 的概念中，数据库分为两类，一类叫master，另一类叫slave。主数据库master进行读写操作，当有数据变化时，会自动同步给slave。slave负责读操作，并接受主数据库同步过来的数据。

一个主数据库可以拥有多个从数据库，而一个从数据库库只能拥有一个主数据库。

4.1.1 原理

  • 复制初始化：当一个从数据库启动后，会向主数据库发送SYNC命令。同时主数据库接收到 SYNC命令后会开始在后台保存快照(即RDB持久化的过程)，并将保存快照期间接收到的命令缓存起来。当快照完成后,Redis 会将快照文件和所有缓存的命令发送给从数据库。从数据库收到后，会载入快照文件并执行收到的缓存的命令。
  • 之后每次主数据库收到写命令时，将会同步给从数据库，从而保证主从数据库一致
